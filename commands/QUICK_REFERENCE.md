## Common Workflows

### New Feature Development
```
/feature-development implement [feature description]
/test-generate [new files]
/security-scan [new code]
/doc-generate [new code]
```

### Code Quality Check
```
/full-review [file or directory]
/performance-analyze [file]
/code-format [file]
```

### Bug Fix
```
/smart-fix [problem description]
/test-generate [fixed code]
```

### Modernization
```
/legacy-modernize [target]
/deps-upgrade all
/test-generate [updated code]
```

## Tips

1. **Be Specific**: Provide detailed requirements for better results
2. **Chain Commands**: Use workflows first, then refine with tools
3. **Review Output**: Always review generated code before committing
4. **Test First**: Run tests after using code generation commands
5. **Security First**: Run security-scan on new code
