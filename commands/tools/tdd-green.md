# TDD Green Tool

Implement minimal code to make tests pass (Green phase of TDD).

## Usage
/tdd-green [function or feature description]

## Example
/tdd-green implement shipping cost calculator to pass tests

## Features

1. **Minimal Implementation**
   - Write just enough code to pass tests
   - Avoid premature optimization
   - Keep implementation simple
   - Focus on test requirements

2. **Code Quality**
   - Clean, readable code
   - Proper error handling
   - Basic validation
   - Simple logic

3. **Test Verification**
   - Ensure all tests pass
   - Verify test coverage
   - Check for regressions

## Output
- Minimal implementation code
- Passing test suite
- Code documentation

## Variables
$ARGUMENTS - Function or feature description

## Implementation Principles
- Make it work first
- Minimal viable code
- No premature optimization
- Focus on test requirements
- Keep it simple
